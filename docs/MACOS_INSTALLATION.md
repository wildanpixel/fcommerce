# macOS Installation Guide

This app is built for macOS through GitHub Actions. Do not use the Windows portable `.exe` on macOS. macOS must use the `.dmg` installer or the packaged `.app` generated by the macOS build job.

## GitHub Download

The macOS artifact is produced by the GitHub Actions workflow:

- Repository: `wildanpixel/fcommerce`
- Workflow: `Build Desktop Artifacts`
- Artifact name: `marketplace-intelligence-os-macos`

GitHub Actions artifacts require access to the repository and a signed-in GitHub account. For a permanent public download URL, publish the generated `.dmg` files through GitHub Releases.

Artifacts are retained for 30 days after each successful workflow run. If the artifact has expired, run the workflow again from GitHub Actions.

## Download From GitHub Actions

1. Open https://github.com/wildanpixel/fcommerce/actions.
2. Select `Build Desktop Artifacts`.
3. Open the latest successful run on the `main` branch.
4. Scroll to `Artifacts`.
5. Download `marketplace-intelligence-os-macos`.
6. Unzip the downloaded artifact.
7. Choose the correct DMG:
   - Apple Silicon Mac, such as M1, M2, M3, or M4: `MarketPlace Keyword Competitor Analysis 1.0.0-arm64.dmg`
   - Intel Mac: `MarketPlace Keyword Competitor Analysis 1.0.0-x64.dmg`

## Create A Fresh macOS Download

Use this when you need a new macOS build without changing source code:

1. Open https://github.com/wildanpixel/fcommerce/actions/workflows/build.yml.
2. Click `Run workflow`.
3. Select branch `main`.
4. Click `Run workflow`.
5. Wait for the `macOS Desktop` job to finish successfully.
6. Download the `marketplace-intelligence-os-macos` artifact from the completed run.

## Install The DMG

1. Double-click the correct `.dmg` file.
2. Drag `MarketPlace Keyword Competitor Analysis.app` into `Applications`.
3. Eject the mounted DMG.
4. Open `Applications`.
5. Control-click `MarketPlace Keyword Competitor Analysis`.
6. Click `Open`.
7. Confirm `Open` again if macOS warns that the developer cannot be verified.

## If macOS Blocks The App

The current build is unsigned and not notarized. This is expected until Apple Developer signing is configured.

Use the normal Gatekeeper flow first:

1. Open `System Settings`.
2. Go to `Privacy & Security`.
3. Find the blocked `MarketPlace Keyword Competitor Analysis` message.
4. Click `Open Anyway`.
5. Confirm `Open`.

If you trust the build and macOS still blocks it, remove the quarantine flag:

```bash
xattr -dr com.apple.quarantine "/Applications/MarketPlace Keyword Competitor Analysis.app"
```

Then open the app again from `Applications`.

## Application Data Location

Runtime data is local to the Mac. Electron stores it under the platform application folder:

```text
~/Library/Application Support/MarketPlace Keyword Competitor Analysis/
```

The app stores the local SQLite database, settings, browser profiles, screenshots, reports, logs, and cache in platform-specific folders. No cloud backend is required.

## macOS Build Configuration

The macOS package is generated by `.github/workflows/build.yml` on a macOS runner with:

```bash
pnpm --filter @marketplace-intelligence-os/desktop exec electron-builder --mac --x64 --arm64
```

The Electron Builder configuration produces:

- `.app` directory build for Intel macOS
- `.app` directory build for Apple Silicon macOS
- `.dmg` installer for Intel macOS
- `.dmg` installer for Apple Silicon macOS

## Future Code Signing

Apple Developer signing and notarization are prepared but not active. When certificates are available, configure them as CI secrets:

- `APPLE_ID`
- `APPLE_APP_SPECIFIC_PASSWORD`
- `APPLE_TEAM_ID`
- `CSC_LINK`
- `CSC_KEY_PASSWORD`

Do not hardcode certificates or Apple credentials in the repository.

## Troubleshooting

- If you downloaded the Windows portable `.exe`, delete it. macOS cannot run it natively.
- If you are on Apple Silicon and the Intel build opens slowly, download the `arm64` DMG instead.
- If the app opens but cannot browse Shopee, complete any Shopee login or verification manually inside the app browser.
- If reports or screenshots are missing, check the app data folder under `~/Library/Application Support/MarketPlace Keyword Competitor Analysis/`.
- If macOS says the app is damaged, remove the quarantine flag with the `xattr` command above and try again.
